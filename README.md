# GLLM
利用`numpy`从0实现LLM
## 训练流程
### 构建词汇表
构建词汇表之前我们需要先读取数据集文件，本项目采用modelscope开源的数据集。
构建词汇表的步骤如下：
1. 分词并统计词频
2. 过滤掉低频词并将剩下的词加入到词汇表
3. 加入特殊tokens，例如`<unk>`,`<pad>`和`<eos>`
3. 保存词汇表
### 创建数据加载器`dataloader`
`dataloader`的主要作用为为模型训练提供`索引2token`和`token2索引`和完美的训练数据格式
前者比较容易，只需要创建两个字典即可
后者需要做如下工作
1. 将所有数据集内容总结到一个**大**列表中，每行数据集在加入大列表时需要加上`<eos>`(end of stence)，告诉模型当预测到`<eos>`时即可停止输出
2. 提供特定`batch_size`和`block_size`的训练文本，因为LLM的预训练是自回归的，它的训练目标是预测当前所有token的下一个token，因此我们的模型输入和标准输出是刚刚创建的大列表的错位
```
x = [2,654,1,34]
y = [654,1,34,46]
```
而`batch_size`是每次进行训练的输入的大小，比如我们一次进行4个输入的训练
```
x = [[3,54,1,53],
    [54,135,632,64],
    [35,5,3,74],
    [76,35,15,63]]
y = [[54,1,53,57],
    [135,632,64,72],
    [5,3,74,86],
    [35,15,63,50]]
```
`block_size`是训练的模型最多可以接受的token总数，比如x = [2,654,1,34,54,2,62,64,634,26]，这个列表一共有10个索引，所以当前模型的block_size只有可怜的10，如果大家有了解过大模型，这个值就是我们平时说的**上下文长度**
### Embedding层
其实到这里我们才算进入真正的LLM
Embedding层又叫做词嵌入层，它的存在是为了将我们上文dataloader输出的索引列表转化为一个真正进行运算的向量组
![embedding_lyaer](assets/imgs/embedding_lyaer.png)

如图，假设现在用户的输入是`The cat sat on The`，输入在经过`dataloader`后会变成`[1,2,5,7,0]`(如图所示)，而图片中间的就是嵌入层(Embedding layer)，可以很轻松的看到，我们在Embedding layer只是做了简单的依据`[1,2,5,7,0]`索引查询到特定的词嵌入向量，并组合成整个的输入

上面这段话有**2个概念**或者说长度需要我们命名，大家可以想一想是什么
我们想一想，首先我们是利用索引`[1,2,5,7,0]`在Embedding layer中查询对不对，那么我们怎么保证索引和Embedding layer的对应关系呢？事实上这个是我们实现规定的，我们在之前训练词汇表时得到了词汇表的大小，而这个大小就是现在我们的Embedding layer的大小，因此我们才可以得到一一对应的嵌入向量。
于是问题又来了，我们现在知道Embedding layer的一个长度是词汇表的大小，另一个长度呢？(因为Embedding layer是一个矩阵，它毫无疑问会有两个长度)，我们想一下另一个长度的作用是什么，用户输入的字符经过dataloader转化为索引列表，再经过Embedding layer索引得到**向量组**，是的，这个向量组是我们真正参与运算的内容，在LLM中它肯定是需要包含一定的语义信息的对吧？那其实不用想也应该越大越好，其实是不是我也不知道，我没做个实验。总的来说，Embedding layer的另一个长度是看自己的，爱多长多长，但我希望它是一个偶数，后续可能有用。(chatGPT-3的这个长度是12288)，对了，这个长度叫做`embed_dim`，即词嵌入维度。
工作原理讲清楚了吗？
我总结一下，用户的输入文本经过dataloader得到词汇表索引，词汇表索引经过Embedding layer得到输入向量组
![alt text](assets/imgs/raw_text2embedding.png)
**！！！忘了很重要的一点！！！**
在语言中，`我爱你`和`你爱我`是完全不同的意思(所以才有暗恋的概念)
那么在模型中，如果我们只是简单的将embedding_lyaer得到的向量组输入模型，它肯定无法分辨这两句话的区别，为此，我们需要人为的为每一个token的向量加上一些固定的**水印**
具体做法是，因为我们模型的上下文长度是固定的，因此我们直接创建一个类似于embedding_lyaer的大矩阵，这个矩阵的长和`embedd_dim`的大小一致(这是显而易见的，因为我们就是要把它加到词嵌入向量),它的宽则是我们的`block_size`，这是因为我们的上下文长度只有那么长！再长的话我们的模型处理不了了！
